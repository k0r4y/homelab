# Purple Team Playbook: T1548.003 - Sudo Privilege Escalation Failure

## Executive Summary
* **Tactic:** Privilege Escalation (TA0004), Defense Evasion (TA0005)
* **Technique:** Abuse Elevation Control Mechanism (T1548)
* **Sub-technique:** Sudo and Sudo Caching (T1548.003)
* **Target System:** `node01` (K3s Cluster Worker Node / Wazuh Agent `002`)
* **Target IP:** `192.168.178.25`
* **Attacker Platform:** Kali Linux VM (Hyper-V External Virtual Switch)
* **Attacker IP:** `192.168.178.29`
* **Status:** Successfully Executed, Detected, and Triaged

## Technical Context & Infrastructure Setup
This validation scenario verifies the SIEM's capability to detect post-exploitation privilege escalation anomalies. Following an initial access event, an adversary attempts to execute administrative actions via the `sudo` binary. The system authentication pipeline routes local authorization failures to the central SIEM to trigger early warning indicators before a root-level compromise occurs.

## Simulation Walkthrough

### 1. Exploitation (Unauthorized Privilege Escalation Attempt)
The operator establishes an active shell session on the target node using the local user account. The operator deliberately inputs incorrect credentials at the system authentication prompt to exhaust the validation attempts and force a system log generation.

```bash
# Establish remote access to the target node from the attacker station
ssh k0r4y@192.168.178.25

# Attempt to execute an administrative shortcut command
# Action: Purposefully enter invalid passwords at the credential prompt
sudo .
```

*Expected Terminal Attack Output:*
```text
[sudo] password for k0r4y: 
Sorry, try again.
[sudo] password for k0r4y: 
Sorry, try again.
[sudo] password for k0r4y: 
sudo: a password is required
```

## Detection Telemetry & Analysis

### Log Source Pipeline
Detection of this attack vector relies on system authentication logs collected natively by the local journal infrastructure.
* **Log Location:** `journald`
* **Collection Engine:** Wazuh Agent log collector engine on `node01` (Agent ID: `002`).
* **Central SIEM Hub:** Wazuh Manager node hosted on `node03`.

### Raw Evidentiary Event Log
The following underlying security event log string was captured natively by the system daemon during execution:

```text
Jun 20 15:34:00 node01 sudo[45223]:    k0r4y : 3 incorrect password attempts ; TTY=pts/0 ; PWD=/home/k0r4y ; USER=root ; COMMAND=.
```

## Strategic Mitigation & Detection Tuning
1. **Compliance Mappings:** This alert triggers regulatory cross-references including **PCI DSS (10.2.4, 10.2.5)** and **GDPR (IV_35.7.d)**, verifying that user access control logging meets auditing standards.
2. **Alert Elevation:** For critical production workloads, you can override standard rule `5404` inside `local_rules.xml` on your manager (`node03`) to route immediate high-priority alerts to monitoring teams if administrative loops break on cluster workers.
