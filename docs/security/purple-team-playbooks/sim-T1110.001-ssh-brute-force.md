# Purple Team Playbook: T1110.001 - SSH Brute Force

## Executive Summary
* **Tactic:** Credential Access (TA0006)
* **Technique:** Brute Force (T1110)
* **Sub-technique:** Password Guessing (T1110.001)
* **Target System:** `node01` (K3s Cluster Worker Node / Wazuh Agent)
* **Target IP:** `192.168.178.25`
* **Attacker Platform:** Kali Linux VM (Hyper-V External Virtual Switch)
* **Attacker IP:** `192.168.178.29`
* **Status:** Successfully Executed, Detected, and Triaged

## Technical Context & Infrastructure Setup
The simulation validates the end-to-end log forwarding telemetry from an enterprise-grade homelab cluster. The infrastructure leverages a centralized Wazuh manager collecting auth events across a decoupled K3s and Hyper-V environment. The attacker machine runs on a bridged local subnet (`192.168.178.0/24`), simulating an internal network threat vector targeting cluster worker infrastructure.

## Simulation Walkthrough

### 1. Reconnaissance & Target Discovery
The attack sequence begins with an adversarial scan using Nmap to identify active management ports, probe for running software versions, and run default NSE (Nmap Scripting Engine) vulnerability scripts against common infrastructure ports.

```bash
# Executing targeted service discovery and vulnerability scripting
# -sV: Probe open ports to determine service/version info
# -sC: Run default Nmap Scripting Engine (NSE) scripts
# -p: Limit the scan to specific target ports (SSH, Web, Monitoring, Metrics)
sudo nmap -sV -sC -p 22,80,443,8080,9100,9090,3000 192.168.178.25
```

### 2. Exploitation & Password Guessing Execution
With port 22 confirmed open, Hydra is launched to conduct an online password guessing attack against the highest-privileged system asset (`root`). The attack utilizes the classic `rockyou.txt` password dictionary, throttled safely to prevent resource denial-of-service while maintaining high visibility.

```bash
# Launching online brute-force tool against the SSH daemon
# -l root: Target the specific username 'root'
# -P [path]: Path to the text file containing the password dictionary list
# ssh://...: Define the target protocol scheme and target host IP address
# -t 4: Restrict to 4 parallel connection tasks to avoid crashing the daemon
# -V: Enable verbose output to print every login/password combination attempt
# -: Read additional options or signal standard parsing parameters
hydra -l root -P /usr/share/wordlists/rockyou.txt ssh://192.168.178.25 -t 4 -V -
```

## Detection Telemetry & Analysis

### Log Source Pipeline
Detection of this attack vector relies on system authentication logs generated natively by the operating system journal daemon.
* **Log Path:** `/var/log/auth.log`
* **Collection Engine:** Wazuh Agent log collector configuration element (`localfile`) on `node01`.

The following underlying security event was written to disk on the host during the automated script execution:

## Strategic Mitigation & Detection Tuning
1. **Threshold Adjustments:** The standard Wazuh rule 5763 relies on a default frequency correlation threshold. To minimize targeted slow-brute-force attacks, tune local rules to flag lower attempts over longer operational intervals.
2. **Active Response:** Implement an automated active response execution routine via `ossec.conf` to automatically trigger local routing blockades via `iptables` or system firewall isolation rules on `node01` upon rule firing.
